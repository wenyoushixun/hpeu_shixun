三层架构：
    Web（表现层，仅仅做数据的接收，页面的跳转）

    Service（服务层，仅仅做与业务相关的逻辑处理）

    Dao（数据交互层，仅仅做与数据库交互的逻辑处理）

MVC 模式：
    Model（模型，一般指传递的数据的实体类，也有说法包含了业务逻辑处理）

    View（视图，指展示到浏览器的页面，现阶段指 jsp 页面）

    Controller（控制器，控制了页面的跳转，在 Struts 中 action 都担任了 Controller 的职责，负责了页面的跳转和数据的传递）

    
    
本案例仅仅作为参考，不可直接使用。JDBC 的实现上只有添加用户功能，其他功能均未实现。
